>是LLM开发的一种框架, 将**NLP(Natural Language Process)自然语言处理**的流程**组件化**

NLP的流程:
1. 读取文本
2. 分割文本
3. 文本向量化(嵌入)
4. 存储向量
5. 接收问题 -- 转换为向量
6. 计算相似度(余弦相似度), 搜索相关文本
7. (AI时代)将返回结果先传递给LLM, 由LLM解析结果
8. 将LLM基于相似度返回的结果的解析果返回给用户

>核心 : 以**链式**的方式, 将多个处理操作**流程化**

# Langchain的局限性
>局限性就在于组件的组织方式是`链式的`

1. 一旦某个组件出错, 无法回退,整个链条都会被破坏, 稳定性太差(耦合度太高)
2. 人工介入困难(除非是自动化和人工两条链路)
3. 工作流程僵化(无法引入循环,分支)
4. 需要手动维护状态(走到哪个节点了, 节点的状态是怎样的)

>解决方式 -- 使用LangGraph

# LangGraph : 面向复杂工作流的图示结构
>`图`结构的组件化流程, 完美克服Langchain链式结构带来的局限性

* 链式流程通常是***线性的、预先定义好***的步骤，难以处理需要循环、分⽀或⻓期状态维护的复杂场 景。
*  此外，在构建多智能体协作、需要⼈⼯介⼊（Human-in-the-loop）或⻓时间运⾏的任务时，需要 更灵活的⼯作流管理和状态持久化⽀持。
