# apt本地配置
> 干什么: 从根据本地配置文件中的地址,发送网络请求,访问对应的网络服务器,获取对应的文件资源
> 为什么:
> 1. 云服务器不需要进行配置,因为厂商已经给你配置好了
> 2. 配置的原因主要是因为网络服务器大部分是国外的.网络请求速度慢




# 进一步理解
>在我不小心执行 rm -rf /*之后, 对自己的服务器执行了重装

重装之后想安装redis,执行`apt search redis` 时却没有检索到,返回如下内容
```shell
Sorting... Done 
Full Text Search... Done
```
后续直接执行`apt install redis`也无法执行成功,询问chatgpt之后,了解到**重装系统之后,需要更新软件源才可以**,执行`apt update`即可更新
apt不会直接从网络中查询程序所在的服务器地址,而是先从本地的软件包索引进行搜索,搜索到之后,才会尝试向服务器请求下载对应的资源
![[Pasted image 20251006224540.png]]
`注意`:apt update != apt upgrade
* update : 更新本地软件索引文件
* upgrade : 依赖本地软件索引文件,对已安装的软件进行升级
