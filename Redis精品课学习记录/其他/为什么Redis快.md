1. Redis数据存储在内存之中,MySQL的数据存储在硬盘上,读取速度不是一个量级的(内存更加接近寄存器)  对比起来更快
2. Redis是单线程+事件驱动,没有线程切换的开销;MySQL是多线程的模型,线程切换,线程阻塞都会放慢读取速度
3. Redis的数据结构设计更加合理,每种存储类型都有专门的编码方式进行存储(string有 raw,int,embstr  在不同场景下实现空间性价比的最优化);MySQL仅仅是通过表结构进行数据的存储,格式更加严格
4. Redis底层使用c写的,更加贴近底层
5. 处理网络IO时,使用了epoll这种多路复用的机制  通过事件驱动的方式大大降低线程因空等导致的资源消耗,提高效率
