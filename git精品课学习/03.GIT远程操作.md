# 向远程仓库推送
push操作是分支和分支之间的操作
查看git cofig


```c
git push origin master:master   建立本地分支和远程分支之间的映射
                本地分支 远程分支
```

![[Pasted image 20251220194535.png]]
**远程仓库的分支要和本地仓库的分支建立连接**
对于master分支, 在clone的时候, 就会直接创建连接

# 拉取远程仓库
```
git pull origin master:master
                远程分支  本地分支
```
![[Pasted image 20251220194608.png]]
```c
git pull = git fetch + git merge
git fetch : 从远程仓库获取最新的提交,分支等更新, 不会更改本地的工作区
git merge :  将获取到的远程分支合并到当前checkout的分支

如果直接git pull, 默认是拉取main分支
```


# 配置git文件
`.gitignore`文件
```c
*.so  // 忽略所有.so文件
!c.so // c.so这个文件不删除
git check-ignore -v d.so  
git add -f filename // 强制进行添加操作
git checkout-ignore -v a.so  查看当前文件被那个屏蔽规则给屏蔽
```

# git标签
`标签`就是对某次commit的**标识**
```c
git tag  查看本地有哪些标签
git tag v1.0 给最新一次提交打标签
git tag -d  删除标签
git tag v1.0 cmid  给指定commit打标签
------------------------------------
git push origin v1.0  向远程推送标签
git push origin --tags  推送所有标签
git push push origin  :v1.0  表示删除远程仓库的标签
```
