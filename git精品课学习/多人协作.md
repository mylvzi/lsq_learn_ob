**master分支是稳定的分支, 一般不推荐直接在master分支上进行操作**
新建分支 -- 在远程创建新的分支, 本地开发者pull一下

解决方案 : 在远程创建分支  在本地创建分支

```c
git branch -r  查看远程分支
git branch -a  查看所有分支

git pull // 直接拉取远程仓库的所有分支
```

>必须本地分支和远程分支进行映射 + 隔离操作


```c
git pull // 获取远程创建的dev分支  还有一层含义, 就是保证本地分支的master是最新的
git checkout -b dev origin/dev  // 在本地创建一个dev分支并和远程的dev分支建立链接
git branch -vv  // 查看本地分支和远程分支的链接关系
-----------------------------------
编辑操作
-----------------------------------
git push origin dev // 自动推送到dev分支  或者直接git push就行
```

一旦建立好链接, 就可以直接git push


# dev分支合并到master分支
>PR单子 : 需要经过审查员的审核操作 -- 实际开发中更推荐!!!


>本地操作 : 本地merge之后, 再进行push操作

```c
// 合并之前先切换到master分支, 保证master分支是最新的
git checkout master
git pull origin master

// 切换到dev分支, 合并master分支(如果有错误,直接在dev分支解决)
git checkout dev
git merge master

// 切换至master分支并推送到远程
git checkout master
git merge dev
git push origin master
```
dev  master : mergedev -- push master

![[Pasted image 20251220172109.png]]

这是在同一分支上进行的操作 -- 十分麻烦 -- 解决冲突是很麻烦的事情
