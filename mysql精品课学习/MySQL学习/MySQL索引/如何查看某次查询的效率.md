这是个非常棒的问题！🔍  
**“MySQL 索引是否真的加快了查询效率”**——可以通过**对比执行计划和查询耗时**来验证。下面我手把手教你验证方法：

---

## ✅ 1. 创建测试表（不建索引）

```sql
CREATE TABLE test_users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(100),
    email VARCHAR(100)
);
```

## ✅ 2. 插入大量数据（模拟真实环境）

```sql
-- 插入10万条数据（用脚本或存储过程更快）
INSERT INTO test_users (username, email)
SELECT
    CONCAT('user', FLOOR(RAND() * 100000)),
    CONCAT('email', FLOOR(RAND() * 100000), '@test.com')
FROM
    information_schema.tables t1,
    information_schema.tables t2
LIMIT 100000;
```

---

## ✅ 3. 运行没有索引的查询，并查看执行计划

```sql
EXPLAIN SELECT * FROM test_users WHERE username = 'user12345';
```

输出中你会看到：

```text
type: ALL       -- 全表扫描
rows: 100000    -- 扫了10万行
```

耗时也会比较高（可以加 `SHOW PROFILE` 或 `SET profiling = 1` 查看详细时间）

---

## ✅ 4. 给 `username` 添加索引

```sql
CREATE INDEX idx_username ON test_users(username);
```

---

## ✅ 5. 再次运行相同查询并查看执行计划

```sql
EXPLAIN SELECT * FROM test_users WHERE username = 'user12345';
```

此时输出可能是：

```text
type: ref        -- 使用了索引
rows: 1          -- 只扫描了极少量行
key: idx_username
```

查询速度明显更快！

---

## ✅ 6. 可选：用 SQL 语句对比耗时

```sql
-- 开启性能分析
SET profiling = 1;

-- 查询一次（有索引）
SELECT * FROM test_users WHERE username = 'user12345';

-- 查看耗时
SHOW PROFILES;
```

---

## ✅ 总结：如何验证索引是否生效？

|验证手段|意义|
|---|---|
|`EXPLAIN`|查看是否使用了索引（`key` 字段）|
|`SHOW PROFILES`|实际耗时对比|
|`rows` 字段|查看扫描行数是否减少|
