我们知道,创建为表的某个列创建索引时可以指定索引的排列顺序,比如
```sql
create index idx_name on user(name desc);
```
这表示对user表中的name列*降序创建了索引*

为什么要有这样的设计呢?主要是为了应对不同的场景

首先我们要知道**索引并不是所有情况下都能提高查询效率的**,索引的执行是先要**扫描创建的索引(映射关系)**,找到对饮的索引列再去定位到对应的行

**索引降序排序的使用场景**
1. 日志系统:查找最近的日志, 按照时间线降序排序
2. 排行榜系统:为**浏览量**列降序创建索引
3. 订单系统能够:查找用户最近购买, 按照时间线降序排序
