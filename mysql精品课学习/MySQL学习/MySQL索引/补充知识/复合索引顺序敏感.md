```sql
create index idx_name_friend on user(name, friend);
```

在查询时如何匹配复合索引呢?
> 先按照name分组, 再去匹配friend

所以匹配的顺序是**从左往右匹配**

如果
```sql
select * from user where friend = 'l';
```
就无法匹配索引, **索引失效**

以下两种场景都能命中
```sql
select * from user where name = 'k';
select * from user where name = 'k' and friend = 'l';
```
